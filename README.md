# DarkWG

VPN-туннель с обфускацией трафика (32-битные заголовки H1-H4, junk-пакеты
Jc/Jmin/Jmax, маскировка размеров S1/S2) + REST API и CLI для управления
пирами. Поднимается одной командой на чистом Ubuntu 24.04.

## Требования

- Ubuntu 24.04 (noble), root-доступ
- Открытый исходящий доступ в интернет на сервере для установки пакетов
- Один свободный UDP-порт для тоннеля (по умолчанию выбирается случайно)

## Установка

```bash
git clone https://github.com/thedarkkness/DarkWG.git
cd DarkWG
sudo DARKWG_ENDPOINT=<домен_или_IP_сервера> bash install.sh
```

Если переменную `DARKWG_ENDPOINT` не задать, скрипт спросит её интерактивно.

После установки:
- интерфейс называется `darkwg0`, конфиг лежит в `/etc/darkwg/darkwg0.conf`
- команды управления — `darkwg show darkwg0`, `darkwg-quick up/down ...`
- подсеть туннеля — `10.13.0.0/16` (до ~65 тысяч клиентов на одну ноду)
- сразу создаётся первый пир: `/etc/darkwg/peers/peer1.conf` + `peer1.png` (QR)
- REST API поднимается на `127.0.0.1:8765` и не торчит наружу

В конце установки скрипт выводит публичный ключ сервера, ключ для API и пути
до файлов первого пира — сохрани их, повторно они не выводятся.

## Подключение клиента

Конфиг (`.conf`) или QR-код из `/etc/darkwg/peers/` можно импортировать в
любое приложение, совместимое с протоколом обфускации AmneziaWG
(параметры Jc/Jmin/Jmax/S1/S2/H1-H4 в конфиге — это его формат). Подходит,
например, официальное приложение AmneziaWG для Android/iOS/Windows/macOS/Linux,
либо собственный White-label клиент, если он у вас есть.

## Управление пирами

**Через CLI (на самом сервере):**

```bash
/opt/darkwg/venv/bin/python3 /opt/darkwg/scripts/darkwg_cli.py add-peer \
  --telegram-user-id 123456789 --ttl-days 30 --out /etc/darkwg/peers/user_123456789

/opt/darkwg/venv/bin/python3 /opt/darkwg/scripts/darkwg_cli.py list-peers

/opt/darkwg/venv/bin/python3 /opt/darkwg/scripts/darkwg_cli.py remove-peer --id 5
```

**Через API (для бота или своей панели):**

Все эндпоинты, кроме `/health`, требуют заголовок `X-API-Key` (значение — в
`/etc/darkwg/api.env` на сервере, в переменной `DARKWG_API_KEY`).

| Метод  | Путь                     | Что делает                                    |
|--------|--------------------------|------------------------------------------------|
| POST   | `/peers`                 | Создать пира, вернуть `.conf` + QR в base64    |
| GET    | `/peers`                 | Список всех пиров                              |
| GET    | `/peers/by-user/{tg_id}` | Пиры конкретного telegram-пользователя         |
| GET    | `/peers/{id}`            | Один пир + актуальный конфиг                    |
| DELETE | `/peers/{id}`            | Отозвать пира                                   |
| POST   | `/peers/{id}/disable`    | Временно выключить                              |
| POST   | `/peers/{id}/enable`     | Включить обратно                                |
| GET    | `/stats`                 | Живая статистика handshake/трафика              |

CLI и API работают с одной и той же базой (`/opt/darkwg/darkwg.db`) — пир,
созданный одним способом, виден через другой.

Пример запроса:

```bash
curl -s -X POST http://127.0.0.1:8765/peers \
  -H "X-API-Key: <ключ из api.env>" \
  -H "Content-Type: application/json" \
  -d '{"telegram_user_id": 123456789, "ttl_days": 30}'
```

## Интеграция с ботом

Шаблон вызова API из aiogram-бота — см. `examples/bot_integration_example.py`.
API слушает только `127.0.0.1`, поэтому бот должен работать на той же машине,
либо нужен отдельный защищённый канал (SSH-туннель/VPN) между ботом и сервером.

## Внутреннее устройство

- Туннель работает на пакете `amneziawg-dkms`/`amneziawg-tools` из официального
  PPA проекта AmneziaWG — это видно через `apt`/`dpkg -l`/`lsmod`. Команды,
  конфиги, имя интерфейса и API в этом репозитории называются `darkwg*` через
  симлинки и собственные systemd-юниты, но сам протокол и обфускация —
  стандартная реализация AmneziaWG, без модификаций ядра.
- Обфускационные параметры (Jc/Jmin/Jmax/S1/S2/H1-H4) генерируются свежими
  случайными значениями при каждой установке — см.
  `scripts/generate_obfuscation_params.py`.

## Хранение приватных ключей

API/CLI хранят приватный ключ клиента в SQLite (`/opt/darkwg/darkwg.db`),
чтобы можно было повторно выдать конфиг без отзыва старого. Файл базы — права
600, доступен только root.
