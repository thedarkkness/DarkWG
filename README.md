# DarkWG

VPN-тоннель с обфускацией трафика (32-битные заголовки H1-H4, junk-пакеты
Jc/Jmin/Jmax, маскировка размеров S1/S2) + REST API и CLI для управления
пирами. Ставится одной командой на чистый Ubuntu 24.04.

## Установка

```bash
git clone <твой репозиторий> darkwg-server
cd darkwg-server
sudo DARKWG_ENDPOINT=node-nl.jawsofplanet.space bash install.sh
```

После установки:
- интерфейс называется `darkwg0`, конфиг лежит в `/etc/darkwg/darkwg0.conf`
- команды управления — `darkwg show darkwg0`, `darkwg-quick up/down ...` (это симлинки на штатный бинарь, поведение идентичное)
- подсеть тоннеля — `10.13.0.0/16` (до ~65 тысяч клиентов на одну ноду)
- сразу создаётся первый пир: `/etc/darkwg/peers/peer1.conf` + `peer1.png` (QR)
- API поднимается на `127.0.0.1:8765`, наружу не торчит

## Управление пирами

**Через CLI (на самом сервере):**

```bash
/opt/darkwg/venv/bin/python3 /opt/darkwg/scripts/darkwg_cli.py add-peer \
  --telegram-user-id 123456789 --ttl-days 30 --out /etc/darkwg/peers/user_123456789

/opt/darkwg/venv/bin/python3 /opt/darkwg/scripts/darkwg_cli.py list-peers

/opt/darkwg/venv/bin/python3 /opt/darkwg/scripts/darkwg_cli.py remove-peer --id 5
```

**Через API (для бота):**

Все эндпоинты, кроме `/health`, требуют заголовок `X-API-Key` (значение — в `/etc/darkwg/api.env`).

| Метод  | Путь                     | Что делает                                    |
|--------|--------------------------|------------------------------------------------|
| POST   | `/peers`                 | Создать пира, вернуть `.conf` + QR в base64    |
| GET    | `/peers`                 | Список всех пиров                              |
| GET    | `/peers/by-user/{tg_id}` | Пиры конкретного telegram-пользователя         |
| GET    | `/peers/{id}`            | Один пир + актуальный конфиг                    |
| DELETE | `/peers/{id}`            | Отозвать пира                                   |
| POST   | `/peers/{id}/disable`    | Временно выключить                              |
| POST   | `/peers/{id}/enable`     | Включить обратно                                |
| GET    | `/stats`                 | Живая статистика handshake/трафика              |

CLI и API работают с одной и той же базой (`/opt/darkwg/darkwg.db`) — пир,
созданный одним способом, виден через другой.

## Технические детали (для тебя, не для клиентов)

- Туннель работает на пакете `amneziawg-dkms`/`amneziawg-tools` из официального
  PPA — это видно через `apt`/`dpkg -l`/`lsmod`, скрыть данный факт на этом уровне
  невозможно средствами конфигурации, и я не стал героически пересобирать
  ядерный модуль ради переименования бинарного символа — риск что-то сломать
  на проде не оправдан результатом (видимый эффект для пользователя/админа
  нулевой что без пересборки, что с ней — команды и так все называются `darkwg*`
  через симлинки).
- Везде, с чем сталкиваются клиенты и с чем ты работаешь день в день — конфиги,
  имя интерфейса, CLI, API, систед-юниты — называется `darkwg`, без единого
  упоминания исходного проекта.
- Обфускационные параметры (Jc/Jmin/Jmax/S1/S2/H1-H4) генерируются свежими
  случайными значениями при каждой установке — см. `scripts/generate_obfuscation_params.py`.

## Хранение приватных ключей

API/CLI хранят приватный ключ клиента в SQLite (`/opt/darkwg/darkwg.db`),
чтобы можно было повторно отдать конфиг без отзыва старого. Файл базы — права 600,
доступен только root.
